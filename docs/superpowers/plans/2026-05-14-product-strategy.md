# ToTra 产品战略：两条主线

> 本文档记录 ToTra 从 KPI 算法平台转向两条核心产品线的战略决策背景、发展路径和终局设想。

---

## 战略背景

### 原方向的根本问题

ToTra 早期以"AI 使用效率 KPI 算法"为核心，用 AIQ/OSS/GTS 三维度衡量员工 AI 使用质量。经过深度分析，该方向存在以下无法解决的根本性问题：

**1. output 质量无法客观量化**
经理用 AI 写文档，无法判断文档质量。工程师用 AI 写代码，无法判断代码质量。唯一勉强可行的是开发者角色（GitHub PR merge rate 可以作为代理指标），但无法覆盖所有岗位。

**2. 算法是代理指标，不是真实效率**
多轮对话 → 得高分，但也可能是在反复纠正 AI 错误。SCU 高 → 得高分，但也可能在刷题。所有维度衡量的是"怎么用 AI"，不是"用 AI 之后产出变好了多少"。

**3. ML 训练是循环定义**
`label_composite` 本身来自 KPI v3 算法。用算法结果训练模型，只能复现算法，无法改进它。真正的 ground truth（产出质量）无法自动采集。

**4. 护城河不成立**
- 纯采纳深度分析：Microsoft Viva、OpenAI Enterprise Dashboard 都在做，且有平台优势
- 行为 proxy 指标：无法说服企业 HR/管理者用于真实绩效决策

### ToTra 真正的资产

在所有 AI 流量的中间，**多厂商、多模型、跨工具**，同时看到员工用 GPT-4、Claude、Gemini 做的所有事情。微软只看 Azure OpenAI，谷歌只看 Vertex AI，没有人在做真正中立的跨厂商企业 AI 基础设施层。

**Gateway 位置 = 真正的护城河起点。**

## 产品线一：AI 治理与合规平台

### 定位

帮企业 IT/法务/合规官回答：**员工用 AI 有没有违规？数据有没有泄露？监管来查能不能交得出来？**

这是刚需，不是可选功能。EU AI Act、GDPR、金融行业 AI 监管正在全球落地。

### ToTra 现有基础（已完成）

| 能力 | 文件 |
|------|------|
| PII 检测中间件（身份证、手机号、银行卡）| `gateway/middleware/pii.go` |
| SHA-256 hash chain 审计日志 | `admin/services/audit.go` + migration 011 |
| 区块链式完整性验证（`/audit-log/verify`）| `admin/api/audit.go` |
| GDPR 数据删除申请 + 保留策略 | migration 012 + `services/data_retention.go` |
| 多租户完全隔离 | 全链路 `tenant_id` |
| IP 白名单（CIDR）| migration 006 + `services/ip_allowlist.go` |
| 配额强制执行（超限 429）| `gateway/middleware/quota.go` |
| Agent 死循环检测（429）| `gateway/middleware/agent.go` |

### 发展路径

**Phase 1（现在 → 3 个月）：把现有能力包装成合规产品**

- 合规事件报告（月度 PDF/CSV）：PII 触发次数、违规用户、未授权模型使用
- 风险评分：按用户、部门、时间段计算合规风险指数
- 实时告警：PII 检测触发时推送飞书/Slack 通知（Bot 通知已有基础）
- Dashboard 合规专区：独立于 KPI 的合规中心页面

**Phase 2（3 → 9 个月）：行业合规包**

- **金融行业**：检测合同金额、账户信息、交易数据进入 AI prompt
- **医疗行业**：检测患者 ID、诊断信息（PHI 合规）
- **通用企业**：SOC 2 Type II AI 附录报告模板
- EU AI Act 合规自检清单（高风险 AI 系统分类）
- 合规审计 API：供企业 SIEM 系统对接（Splunk、Microsoft Sentinel）

**Phase 3（9 → 24 个月）：合规智能化**

- 异常行为检测：识别员工绕过公司 AI 政策的模式
- 跨公司合规 benchmark：匿名化后的行业合规事件频率对比
- 合规咨询 marketplace：连接合规顾问与企业（平台抽佣）
- 监管动态订阅：新法规落地自动推送影响评估

### 终局

**成为企业 AI 合规的基础设施层。**

类比 Workday 之于 HR 系统——不是最性感的产品，但是每家企业不得不有的东西。每新出一条 AI 监管法规，就是一次自动续费理由。

护城河成熟后的形态：
- 积累的审计历史让客户无法迁移（合规记录断档 = 监管风险）
- 跨厂商中立地位让大厂无法复制（微软不会帮你审计谷歌的流量）
- 行业合规包形成垂直壁垒（金融、医疗客户迁移成本极高）
- 监管机构认可 ToTra 作为 AI 审计存证工具（终极护城河）

## 产品线二：AI 成本优化平台

### 定位

帮企业 CFO/IT 回答：**AI 工具的钱花在哪了？哪些钱是浪费的？怎么在不降质量的前提下省钱？**

价值主张直接：**帮你省下的钱比订阅费多。**

### ToTra 现有基础（已完成）

| 能力 | 文件 |
|------|------|
| 全量请求记录（模型、token、SCU、用户、时间）| `usage_records` 表 |
| SCU 计费引擎（多模型折算）| `gateway/middleware/quota.go` |
| 部门成本分摊报告 + CSV 导出 | `admin/services/ops_reports.go` |
| 月末预算预测（线性外推）| `admin/services/ops_reports.go` |
| 低活跃员工识别 | `admin/services/ops_reports.go` |
| Executive ROI Reports（季度 + 行业基准）| `admin/services/roi.go` |
| 多模型配置管理 | `admin/services/model_configs.go` |

### 发展路径

**Phase 1（现在 → 3 个月）：成本可视化**

- 实时成本 Dashboard：今日/本周/本月 AI 支出，按部门/用户/模型分层
- 浪费识别：
  - 重复请求检测（同一用户短时间内高度相似的请求）
  - 非工作时间异常消耗
  - 超配模型使用（GPT-4 处理简单摘要任务）
- 成本预警：月度预算超支预警，推送到管理员

**Phase 2（3 → 9 个月）：智能路由与优化**

- **模型自动路由**：根据请求复杂度自动选择最合适的模型
  - 简单问答 → GPT-4o-mini / Claude Haiku（便宜 10 倍）
  - 复杂推理 → GPT-4o / Claude Sonnet
  - 代码生成 → 专用 Code 模型
- **语义缓存**：相似请求命中缓存，不重复调 API（Redis 已有基础）
- **Prompt 压缩**：自动检测冗余上下文，减少 input token 消耗
- 优化建议报告：每月给 CFO 发"可节省 X 元"报告，附具体措施

**Phase 3（9 → 24 个月）：跨公司成本 Benchmark**

- 行业成本基准：「和你规模相近的金融公司，人均月 AI 支出中位数是 $42，你是 $89」
- 模型性价比排行：基于真实企业用量（匿名化）的模型 ROI 对比，比官方 benchmark 更贴近企业场景
- 采购谈判支持：用平台数据支撑企业与 OpenAI/Anthropic 谈量价折扣
- AI 预算规划工具：根据历史数据和增长预测，帮企业制定 AI 年度预算

### 终局

**成为企业 AI 支出的智能优化层。**

类比 Datadog 之于云监控——工程师/IT 不会手动优化每一个 AI 调用，他们需要一个平台自动做这件事。

护城河成熟后的形态：
- **数据网络效应**：客户越多 → benchmark 越准 → 路由越智能 → 省钱越多 → 客户越多
- **模型路由黑盒**：积累的路由决策数据是竞争对手无法复制的资产
- **CFO 级信任**：成为企业 AI 支出的单一 source of truth，预算决策依赖它
- **与 AI 厂商谈判筹码**：掌握真实用量数据的平台，有资格代理企业与 OpenAI/Anthropic 谈集体议价

## 两条产品线的协同

### 为什么要同时做

| | 单做合规 | 单做成本优化 | 两个一起 |
|--|---------|------------|---------|
| Gateway 价值 | 部分发挥 | 部分发挥 | 完全发挥 |
| 客户黏性 | 高（合规锁定）| 中（省钱但可切换）| 极高 |
| 销售切入点 | 法务/IT | CFO/IT | 同一个 IT 负责人 |
| 数据复用 | 只用合规数据 | 只用成本数据 | 共享全量流量数据 |

两条线面向同一个买单人（企业 IT 负责人），卖的是同一个 gateway 接入，数据完全共享，边际成本极低。

### 统一平台架构

```
员工的 AI 工具（Cursor / ChatGPT / Claude.ai / 内部系统）
                    ↓
            ToTra Gateway（唯一接入点）
                    ↓
        ┌───────────┴───────────┐
        │                       │
   合规引擎                  成本引擎
   PII 检测                  token 计量
   审计日志                  模型路由
   风险评分                  缓存命中
        │                       │
        └───────────┬───────────┘
                    ↓
            统一 Dashboard
        合规中心 ｜ 成本中心 ｜ 管理后台
```

### 销售策略

- **合规作为敲门砖**：用"帮你应对 AI 监管"打开企业大门（法务天然支持，阻力小）
- **成本优化作为续费理由**：「上个月帮你省了 $3,200，订阅费 $500」，CFO 主动续费
- **两条线互相强化**：合规客户天然成为成本优化客户，因为数据已经在流经 gateway

## 与原 KPI 算法的关系

原有 KPI 算法（AIQ/OSS/GTS）、Leaderboard、AI Fuel 奖励系统等功能**不删除**，作为可选模块保留：

- 对于想做员工 AI 采纳激励的企业，KPI 模块仍有价值（「谁用 AI 最积极」）
- 重新定位为**采纳深度追踪**，不再声称衡量效率
- ML Infrastructure（Ridge 训练）暂停主动推进，等合规/成本两条线积累到足够真实数据后重新评估是否有训练价值
